# 基于昇腾边缘计算的人体跌倒检测与报警系统

一个面向独居老人、病房和安全监护场景的边缘侧跌倒检测系统。项目覆盖数据处理、YOLOv8 模型训练、指标评估、昇腾 OM 模型推理、视频检测、RTMP 推流和 MQTT 告警上报等流程。

> 该仓库主要展示深度学习目标检测、边缘 AI 部署、视频流处理和告警联动能力。

## 功能特性

- YOLOv8 跌倒检测模型训练、验证与测试。
- 支持图像、视频和摄像头实时检测。
- 支持昇腾 NPU 加载 OM 模型进行边缘推理。
- 融合 MediaPipe Pose 关键点进行姿态与时序判断。
- 通过状态机降低单帧误判导致的误报。
- 使用 MQTT 上报跌倒事件和设备在线状态。
- 使用 FFmpeg 将检测画面编码为 H.264 并通过 RTMP 推送。
- 提供 PyQt5 可视化工具，用于训练、测试、推理和数据集转换。

## 技术栈

- 开发语言：Python
- 深度学习：PyTorch、Ultralytics YOLOv8
- 边缘推理：Ascend NPU、CANN、ais_bench
- 视觉处理：OpenCV、MediaPipe
- 可视化工具：PyQt5
- 通信与推流：MQTT、FFmpeg、RTMP
- 辅助工具：NumPy、psutil

## 检测流程

```text
摄像头 / 视频
      │
      ▼
昇腾 NPU / YOLOv8 人体检测
      │
      ▼
MediaPipe 姿态关键点分析
      │
      ▼
多特征投票 + 滑动窗口 + 状态机
      ├── 正常：持续监测
      └── 跌倒：MQTT 告警 + RTMP 视频推送
```

## 快速开始

### 1. 创建 Python 环境

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux
source .venv/bin/activate
```

### 2. 安装基础依赖

```bash
pip install opencv-python mediapipe numpy paho-mqtt psutil
```

如需重新训练模型，请根据自己的 CUDA/PyTorch 环境安装 `torch`、`ultralytics` 等训练依赖。

### 3. 昇腾部署依赖

在昇腾边缘设备上运行 OM 模型推理前，需要准备：

- 对应硬件版本的驱动与固件
- CANN 开发/运行环境
- `ais_bench`
- 已转换完成的 `.om` 模型文件

同时需要根据实际部署环境修改模型路径、MQTT Broker、RTMP 地址和摄像头输入源。

### 4. 运行检测

```bash
python FALL_DOWN.py
```

## 模型评估记录

项目保留了训练结果和评估图表。当前公开记录中，`train2` 训练结果最高 mAP50 约为 **0.958**，对应 mAP50-95 约为 **0.654**。实际指标会受到数据集规模、标注质量、训练参数和模型结构影响。

公开脚本默认目标帧率为 **15 FPS**；实际帧率与昇腾型号、OM 模型、视频分辨率、预处理开销和推流配置有关，部署时建议在目标硬件上单独压测。

## 设计亮点

- 基于 YOLOv8 完成人体/跌倒目标检测，并保留 CBAM 等模型配置与训练实验结果。
- 融合人体框、姿态关键点、躯干高度比例、肩胯角度、头部位置、垂直速度和身体倾斜角等多特征。
- 使用 `NORMAL → FALLING → FALLEN → RISING` 状态机，结合滑动窗口投票、持续时间与恢复判断降低误报。
- 检测事件通过 MQTT 上报，可对接移动端通知、云端记录或本地告警系统。
- 检测画面通过 FFmpeg/RTMP 推送，便于远程查看和回放。
